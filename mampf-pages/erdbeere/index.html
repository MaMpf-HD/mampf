Beispieldatenbank

Die Seite „Beispiel-Datenbank“ ist eine Unterseite einer Veranstaltung, zu der wiederum weitere Unterseiten gehören. Sie ermöglicht Nutzer*innen die Suche nach konkreten Beispielen. Dazu geben Nutzer*innen an, welche Eigenschaften erfüllt und verletzt sein sollen. Als Suchergebnis werden Beispiele oder ein Beweis, dass ein solches Objekt nicht existieren kann, ausgegeben.

*Links ergänzen*

Navigation zu dieser Seite​

Um auf die Beispieldatenbank zuzugreifen, öffnet man zunächst eine Veranstaltungsseite. Dazu klickt man auf das Logo oben links in der Navigationsleiste, um auf die persönliche Startseite zu gelangen. Dort wählt man die gewünschte Veranstaltung aus. Auf der Veranstaltungsseite klickt man dann auf den Eintrag „Beispiel-Datenbank“ in der Seitenleiste auf der linken Seite.

Bereiche der Seite​

Die Seite „Beispiel-Datenbank“ gliedert sich in vier Teilbereiche: die eigentliche Seite „Beispiel-Datenbank“, die Navigationsleiste, die Seitenleiste zur Navigation innerhalb einer Veranstaltung und den Footer. Bedienelemente der und mögliche Aktionen auf Unterseiten​

Die Beispieldatenbank besteht aus den Unterseiten „Startseite“, „Beispielsuche“, „Suchergebnis“, „Struktur“, „Eigenschaft“, und „Beispiel“. Die Bedienelemente aller dieser Seiten werden nun beschrieben. Das folgende Schaubild zeigt die Navigationsmöglichkeiten zwischen den Unterseiten der Beispieldatenbank.

Startseite​



  • Wechsel zur Seite „Beispielsuche“.
  • (Überschrift der Box, z.B. ) Wechsel zur Seite „Struktur“.
  • (in der Box „Struktur“, z.B. in der Box „kommutativer Ring mit Eins“) Wechsel zur Seite „Eigenschaft“.

Beispielsuche​



  • Wechsel zur Seite „Strukturenübersicht“.
  • (z.B. ) Wechsel zur Seite einer anderen Struktur.
  • Führe eine Suche mit den ausgewählten Anforderungen an das Beispiel durch.
  • bzw. Aktiviere bzw. deaktiviere die globale Suche.
  • (in der Box „Erfüllt“ oder „Verletzt“, z.B. ) Wähle eine Eigenschaft aus, die das Beispiel (nicht) aufweisen soll. Hier kann höchstens eine Eigenschaft ausgewählt werden.

Erfolgreiche Suche​



  • (in der Box „Suchparameter“ unter „Erfüllt“ oder „Verletzt“, z.B. ) Wechsel auf die Seite der Eigenschaft.
  • (in der Box „Direkte Treffer“, z.B. ) Wechsel auf die Seite des Beispiels.

Erfolglose Suche​



  • (in der Box „Suchparameter“ unter „Erfüllt“ oder „Verletzt“, z.B. ) Wechsel auf die Seite der Eigenschaft.
  • (in der Box „Widerspruch“, z.B. ) Wechsel auf die Seite der Eigenschaft.

Struktur​



  • Wechsel zur Seite „Beispielsuche“.
  • Wechsel zur Seite „Strukturenübersicht“.
  • (in der Box „Struktur“ unter „Verknüpfte Begriffe“, z.B. in der Box „kommutativer Ring mit Eins“) Gehe auf die Seite des Begriffs.
  • (in der Box „Mögliche Eigenschaften“, z.B. ) Wechsel auf die Seite der Eigenschaft.
  • (in der Box „Beispiele“, z.B. ) Wechsel auf die Seite des Beispiels.

Eigenschaft​



  • Wechsel zur Seite „Beispielsuche“.
  • Wechsel zur Seite „Strukturenübersicht“.
  • (in der Box „Eigenschaft von Struktur“ unter „Verknüpfte Begriffe“, z.B. in der Box „Eigenschaft ‚faktoriell‘ von kommutativer Ring mit Eins“) Gehe auf die Seite des Begriffs.
  • (in der Box „Beispiele (Eigenschaft erfüllt)“ bzw. „Beispiele (Eigenschaft verletzt)“, z.B. ) Wechsel auf die Seite des Beispiels.

Beispiel​



  • Wechsel zur Seite „Beispielsuche“.
  • Wechsel zur Seite „Strukturenübersicht“.
  • (Überschrift der Box „Beispiel für Struktur“, z.B. in „Beispiel Z[X] für kommutativer Ring mit Eins“)
  • (in der Box „Erfüllte Eigenschaft“ bzw. „Verletzte Eigenschaft“, z.B. ) Wechsel auf die Seite der Eigenschaft.

Hinweis für Editor*innen​

Damit die Beispieldatenbank in der Seitenleiste einer Veranstaltung angezeigt wird, muss sie zuvor auf der Seite „Veranstaltung bearbeiten“ freigeschaltet werden. Dort kann auch festgelegt werden, welche Strukturen verfügbar sein sollen.
